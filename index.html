CH — 01

Getting Started with Excel

What is Excel?

Microsoft Excel is a spreadsheet application used to store, organise, analyse, and visualise data. It works in a grid of rows and columns where every intersection is called a cell.

📁

Workbook

The .xlsx file itself — like a folder that holds all your sheets and data.

📄

Worksheet

Each tab at the bottom (Sheet1, Sheet2) — a single grid of rows and columns.

🔲

Cell

The box at the intersection of a row and column. e.g. B3 = column B, row 3.

📐

Range

A block of cells — e.g. A1:D10 means from cell A1 down to D10.

Cell Reference Types

When you copy a formula to other cells, how the references behave depends on their type. This is one of the most important concepts in Excel.

Cell Reference Types
A1 -- Relative: row and column both shift when you copy $A$1 -- Absolute: stays fixed no matter where you copy it $A1 -- Mixed: column A is locked, row shifts A$1 -- Mixed: row 1 is locked, column shifts
Pro Tip: Press F4 while your cursor is inside a reference to cycle through all four types automatically.

Referencing Another Sheet

Cross-sheet references
=Sheet2!B3 -- Value of cell B3 on Sheet2 =Sales!A1:D100 -- Range on the sheet named "Sales"

Data Types & Formatting

Excel recognises different types of data and handles each differently. Getting the data type right is essential — wrong types break formulas silently.

The Core Data Types

TypeHow to recognise itExamples
NumberRight-aligned by default42, 3.14, -100
Text (String)Left-aligned by default"Alice", "Delhi", "Male"
Date / TimeRight-aligned, shown as a date01/01/2024, 14:30
BooleanCentred, all capsTRUE, FALSE
ErrorStarts with ##N/A, #VALUE!, #DIV/0!

Custom Number Formats

The same number (e.g. 40000) can display many ways. Open Ctrl+1 → Number → Custom and type a format code:

Format Codes
"#,##0" → 40,000 (thousands separator) "#,##0.00" → 40,000.00 (2 decimal places) "₹#,##0.00" → ₹40,000.00 (currency symbol) "0%" → 40% (multiplies value by 100) "dd-mmm-yyyy" → 01-Jan-2024 (date format) "hh:mm AM/PM" → 02:30 PM (time format) "[Green]0;[Red]-0" → positive = green, negative = red

Converting Between Types

Type Conversion
=VALUE(A1) -- Text "123" → number 123 =TEXT(A1, "dd-mmm-yyyy") -- Date/number → formatted text string =DATEVALUE("01-Jan-2024") -- Date-as-text → real date serial number =N(A1) -- TRUE→1, FALSE→0
Common trap: Numbers stored as text look fine but SUM() and AVERAGE() silently ignore them. Fix using Data → Text to Columns or wrap in =VALUE().
Quick Quiz
Cell A1 shows 40,000 and B1 shows 40000. Are they the same value?
Yes — formatting only changes the display. Both cells contain the number 40000.
You import a CSV and dates appear left-aligned as 01/04/2024. What is the problem?
Left-aligned means Excel stored them as text, not real dates. Use =DATEVALUE(A1) or Data → Text to Columns to fix.

Working with Tables

Excel Tables (Ctrl+T) give your data structure, a name, and automatic behaviour. Always convert raw data into a Table.

Creating a Table

1
Select your data

Click anywhere inside it. Make sure the first row has column headers.

2
Press Ctrl+T or go to Insert → Table

Confirm "My table has headers" and click OK.

3
Name your Table

Table Design tab → Table Name box → type a name like tblUsers.

Structured References

Table Column References
=SUM(tblUsers[Salary]) -- Sum the entire Salary column =tblUsers[@Salary] -- @ = current row's Salary =tblUsers[[Name]:[City]] -- Range from Name column to City column
Why use Tables? They auto-expand when you add rows, formula references update automatically, filters and sorting are built in, and the Total Row gives instant aggregates.

Sorting & Filtering

AutoFilter
Click the dropdown arrow on any column header to: Sort A → Z or Z → A Filter by specific values Filter by condition (e.g. "Greater Than 50000") Filter by colour -- Clear all filters: Data → Clear -- Keyboard: Alt + Down Arrow to open the dropdown

Formulas & Operators

Every Excel formula starts with =. Without it, Excel treats the entry as plain text.

Arithmetic Operators

Arithmetic
=A1+B1 -- Addition =A1-B1 -- Subtraction =A1*B1 -- Multiplication =A1/B1 -- Division (⚠ shows #DIV/0! if B1 = 0) =A1^2 -- Exponent: A1 squared =MOD(A1,2) -- Remainder: MOD(7,2) = 1

Comparison Operators

Comparison — returns TRUE or FALSE
=A1=B1 -- Equal to =A1<>B1 -- Not equal to =A1>B1 -- Greater than =A1<B1 -- Less than =A1>=B1 -- Greater than or equal =A1<=B1 -- Less than or equal

Text Joining

Concatenation
=A1&" "&B1 -- "Nitin" & " " & "Mehra" → "Nitin Mehra" =CONCAT(A1," ",B1) -- Same result using the CONCAT function

Order of Operations

Precedence Examples
=2+3*4 → 14 (multiply first: 3×4=12, then +2) =(2+3)*4 → 20 (parentheses override order) =10/2^2 → 2.5 (exponent first: 2²=4, then 10÷4)
Quick Quiz
What is the result of =5+2*3?
11 — multiplication (2×3=6) runs before addition (5+6=11).
Write a formula to check if the value in A1 is above 60000.
=A1>60000 → returns TRUE or FALSE. Inside an IF: =IF(A1>60000,"High","Low")
CH — 05

Lookup Functions

Lookup functions find a value in one list and return related data from another — like looking up a student's roll number to find their score.

VLOOKUP

Searches for a value in the leftmost column of a range and returns a value from the same row in a column you specify.

VLOOKUP
=VLOOKUP(lookup_value, table_range, col_number, 0) -- ^ ^ ^ ^ -- what to find where to look which column exact match -- Find the salary of employee whose ID is in A2: =VLOOKUP(A2, $D$2:$F$100, 3, 0) -- Last argument: 0 = exact match (always use 0)
Limitation: VLOOKUP can only look to the right — the search column must be the leftmost column of your range. Use INDEX+MATCH or XLOOKUP to look left.

INDEX + MATCH

MATCH finds the row position of a value. INDEX fetches the value at that position. Together they can look in any direction.

INDEX + MATCH
=INDEX(return_range, MATCH(lookup_value, search_range, 0)) -- Find salary for employee named "Alice": =INDEX(C2:C100, MATCH("Alice", A2:A100, 0)) -- MATCH alone returns the row number: =MATCH("Alice", A2:A100, 0) -- e.g. returns 5 -- Two-way lookup (match both row and column): =INDEX($B$2:$F$20, MATCH(A2,$A$2:$A$20,0), MATCH(B1,$B$1:$F$1,0))

XLOOKUP — Excel 365 / 2021+

Replaces both VLOOKUP and HLOOKUP. Simpler syntax, any direction, returns a custom value when nothing is found.

XLOOKUP
=XLOOKUP(A2, tblUsers[ID], tblUsers[Salary], "Not Found") -- Return multiple columns at once (results spill automatically): =XLOOKUP(A2, tblUsers[ID], tblUsers[[Name]:[City]])
Which Lookup Should I Use?
FeatureVLOOKUPINDEX+MATCHXLOOKUP
Look directionRight onlyAnyAny
Not found result#N/A error#N/A errorCustom value
Return multiple columnsNoNoYes (spills)
AvailabilityAll versionsAll versions365 / 2021+
Quick Quiz
Your search column is column D and you need a value from column B (to the left). Which function cannot do this?
VLOOKUP — it only looks to the right. Use INDEX+MATCH: =INDEX(B:B, MATCH(A2, D:D, 0)) or XLOOKUP: =XLOOKUP(A2, D:D, B:B)
CH — 06

Logical Functions

Logical functions test conditions and return different values depending on whether the condition is true or false.

IF — The Foundation

IF
=IF(B2>70000, "High", "Low") -- Nested IF (multiple conditions in sequence): =IF(B2>90000, "Senior", IF(B2>70000, "Mid", IF(B2>50000, "Junior", "Intern")))

IFS — Cleaner Alternative (Excel 2019+)

IFS
=IFS(B2>90000, "Senior", B2>70000, "Mid", B2>50000, "Junior", TRUE, "Intern") -- TRUE = the else / default

AND, OR, NOT

Combining Conditions
-- AND: ALL conditions must be true =IF(AND(B2>60000, C2="Delhi"), "Match", "No") -- OR: at least ONE condition must be true =IF(OR(C2="Male", C2="Other"), "Yes", "No") -- NOT: reverses the result =IF(NOT(C2="Female"), "Non-female", "Female")

IFERROR, IFNA & SWITCH

Error Handling
-- IFERROR: catches any error =IFERROR(VLOOKUP(A2,D:F,3,0), "Not Found") -- IFNA: catches only #N/A (more precise) =IFNA(XLOOKUP(A2,D:D,F:F), "No match") -- SWITCH: match one value against multiple exact options =SWITCH(C2, "Male","M", "Female","F", "Other","O", "Unknown")
CH — 07

Text Functions

Text functions help you clean, extract, and combine text data — essential for working with imported or messy datasets.

Length, Case & Cleanup

LEN / LOWER / UPPER / PROPER / TRIM / CLEAN
=LEN(A1) -- Count characters: "Nitin" → 5 =LOWER(A1) -- "NITIN" → "nitin" =UPPER(A1) -- "nitin" → "NITIN" =PROPER(A1) -- "nitin mehra" → "Nitin Mehra" =TRIM(A1) -- Remove leading, trailing & extra internal spaces =CLEAN(A1) -- Remove invisible non-printable characters

Extracting Parts of Text

LEFT / RIGHT / MID
=LEFT(A1, 3) -- "Nitin" → "Nit" =RIGHT(A1, 3) -- "Nitin" → "tin" =MID(A1, 2, 3) -- Start pos 2, take 3 chars → "iti" -- Extract domain from email: =MID(A1, FIND("@",A1)+1, LEN(A1)) -- "nitin@example.com" → "example.com"

Finding & Replacing

FIND / SEARCH / SUBSTITUTE
-- FIND: case-sensitive, returns position number =FIND("@", "nitin@example.com") → 6 -- SEARCH: case-insensitive, supports wildcards =SEARCH("nitin", "Nitin Mehra") → 1 -- SUBSTITUTE: replace specific text =SUBSTITUTE(A1, "Ltd", "Limited") -- Remove all spaces: =SUBSTITUTE(A1, " ", "")

Joining Text

CONCAT / TEXTJOIN
=CONCAT(A1, " ", B1) -- Join two cells -- TEXTJOIN: join a whole range with a separator =TEXTJOIN(", ", TRUE, A1:A10) -- Result: "Alice, Bob, Charlie, David"
Quick Quiz
Cell A1 has " Nitin Mehra ". What does =TRIM(A1) return?
"Nitin Mehra" — TRIM removes all leading/trailing spaces and collapses multiple internal spaces into one.
Write a formula to extract the first name from "Nitin Mehra" in A1.
=LEFT(A1, FIND(" ", A1) - 1) → finds space at position 6, then LEFT(5) = "Nitin"
CH — 08

Date & Time Functions

Excel stores dates as serial numbers (1 = January 1, 1900). This means you can do arithmetic with dates — subtract two dates to get days between them.

Today & Now

TODAY / NOW
=TODAY() -- Today's date. Updates every day automatically. =NOW() -- Current date + time. Updates on recalculation.

Extracting Date Parts

YEAR / MONTH / DAY / WEEKDAY
=YEAR(A1) -- 2024 =MONTH(A1) -- 4 (April) =DAY(A1) -- 15 =WEEKDAY(A1, 2) -- 1=Mon … 7=Sun (type 2 = Monday start) =HOUR(A1) -- Extract hour from a time value =MINUTE(A1) -- Extract minute

Date Arithmetic

DATEDIF / DAYS / EDATE / EOMONTH
-- DATEDIF: difference between two dates =DATEDIF(A1, TODAY(), "Y") -- Age in complete years =DATEDIF(A1, TODAY(), "M") -- Complete months =DATEDIF(A1, TODAY(), "D") -- Days between -- DAYS: simple days difference =DAYS(TODAY(), A1) -- Days since date in A1 -- EDATE: shift by N months =EDATE(TODAY(), 3) -- 3 months from today =EDATE(TODAY(), -1) -- 1 month ago -- EOMONTH: last day of a month =EOMONTH(TODAY(), 0) -- Last day of current month =EOMONTH(TODAY(), 1) -- Last day of next month

Working Days

NETWORKDAYS / WORKDAY
=NETWORKDAYS(A1, B1) -- Working days between two dates =NETWORKDAYS(A1, B1, HolidayRange) -- Also excludes specified holidays =WORKDAY(TODAY(), 30) -- Date that is 30 working days from today
CH — 09

Aggregate Functions

Aggregate functions summarise a range of values into a single number. The conditional versions let you aggregate only the rows that match a condition.

Basic Aggregates

SUM / AVERAGE / COUNT / MIN / MAX
=SUM(B2:B100) -- Total of all values =AVERAGE(B2:B100) -- Mean (average) =COUNT(B2:B100) -- Count cells with numbers only =COUNTA(A2:A100) -- Count non-empty cells (includes text) =COUNTBLANK(A2:A100) -- Count empty cells =MIN(B2:B100) -- Smallest value =MAX(B2:B100) -- Largest value =LARGE(B2:B100, 2) -- 2nd largest value =SMALL(B2:B100, 3) -- 3rd smallest value

Conditional Aggregates

SUMIF / COUNTIF / AVERAGEIF
-- Sum salaries for Female employees only: =SUMIF(C2:C100, "Female", B2:B100) -- Count employees earning above 60000: =COUNTIF(B2:B100, ">60000") -- Wildcard: names starting with A: =COUNTIF(A2:A100, "A*") -- Names containing "kumar": =COUNTIF(A2:A100, "*kumar*")

Multi-Condition — SUMIFS / COUNTIFS

SUMIFS / COUNTIFS
-- Sum salaries for Female employees earning above 60000: =SUMIFS(B2:B100, C2:C100, "Female", B2:B100, ">60000") -- Count Male employees in Delhi: =COUNTIFS(C2:C100, "Male", D2:D100, "Delhi")
Quick Quiz
Write a SUMIFS to add salaries where gender (col C) = "Female" AND salary (col B) > 70000.
=SUMIFS(B2:B100, C2:C100, "Female", B2:B100, ">70000") — note the S at the end, it's SUMIFS not SUMIF.

Data Validation

Data Validation restricts what can be typed into a cell — preventing errors, creating dropdowns, and enforcing rules on input data.

Setting Up

1
Select the cells to validate
2
Data → Data Validation → Data Validation…

The dialog has three tabs: Settings, Input Message, Error Alert.

3
Set your rule in Settings

Pick validation type, define the criteria, and optionally add messages.

Validation Rules

Examples
-- Whole number between 18 and 60 (age): Allow: Whole number | Between | Min: 18 | Max: 60 -- Dropdown list: Allow: List | Source: Male,Female,Other -- Or reference a named range: Source: $G$2:$G$10 -- Date range: Allow: Date | Between | 01/01/1960 | 31/12/2005 -- Custom formula (most powerful): =LEN(A1)<=10 -- max 10 characters =ISNUMBER(A1) -- must be a number =COUNTIF($A:$A,A1)=1 -- no duplicates allowed
Dependent Dropdowns: Name each category's list (e.g. name a range "Maharashtra"), then set the second dropdown source to =INDIRECT(A1) where A1 holds the category name.
Limitation: Validation only blocks direct typing. Pasted data can still bypass it. Use VBA for stricter enforcement.

Named Ranges

A Named Range gives a friendly name to a cell or range. Instead of writing $B$2:$B$100 everywhere, you write SalaryData. Formulas become readable and easy to update.

Creating Names

3 Ways to Create
-- Method 1: Name Box (fastest) Select range → click Name Box (top-left) → type name → Enter -- Method 2: Formulas → Define Name Name: SalaryData Scope: Workbook Refers to: =Sheet1!$B$2:$B$100 -- Method 3: Formulas → Create from Selection Select data + headers → tick "Top row" → creates a name per column

Using Named Ranges

In Formulas
-- Without name: =SUM($B$2:$B$100) -- With named range: =SUM(SalaryData) -- much cleaner! =VLOOKUP(A2, UserTable, 3, 0) -- Dynamic (auto-expands as data grows): Refers to: =OFFSET(Sheet1!$A$2, 0, 0, COUNTA(Sheet1!$A:$A)-1, 1)
Name Manager: Formulas → Name Manager (Ctrl+F3) — view, edit, and delete all names. Use "Filter → Names with Errors" to clean up broken references.

PivotTables

PivotTables are Excel's most powerful analysis tool — summarise, group, and explore thousands of rows in seconds, entirely with drag and drop.

Creating a PivotTable

1
Click inside your Table or data range
2
Insert → PivotTable → New Worksheet
3
Drag fields into the four areas

Rows — what to group by  |  Values — what to calculate  |  Columns — split into sub-categories  |  Filters — top-level filter

Common PivotTable Setups
-- Average salary by gender: Rows → Gender | Values → Salary (set to Average) -- Count of employees by gender: Rows → Gender | Values → Name (set to Count) -- Only show genders where average salary > 75000: Right-click a value → Filter → Value Filters → Avg Salary → Greater Than 75000 -- Refresh when source data changes: Right-click inside PivotTable → Refresh (or Alt+F5)

Grouping

Date & Number Grouping
-- Group dates by Month/Quarter/Year: Right-click a date cell → Group → select Years, Quarters, Months -- Group numbers into salary bands: Right-click a number → Group Starting at: 40000 | Ending at: 100000 | By: 10000 → Creates: 40000-50000, 50001-60000…

Calculated Fields

Add a custom column to your PivotTable
PivotTable Analyze → Fields, Items & Sets → Calculated Field Name: Tax Formula: = Salary * 0.30
Quick Quiz
You want to see total salary per city, but only cities where the total exceeds ₹5,00,000. Where do you set this?
Right-click the City field → Filter → Value Filters → Sum of Salary → Greater Than → 500000.

Advanced Formulas

Dynamic Array functions (Excel 365+) spill results into multiple cells automatically. SUMPRODUCT works in all versions and is incredibly versatile.

FILTER — Dynamic WHERE Clause

FILTER
-- Return only Female rows (results spill automatically): =FILTER(A2:D100, C2:C100="Female", "No results") -- AND condition (multiply booleans): =FILTER(A2:D100, (C2:C100="Female")*(B2:B100>60000)) -- OR condition (add booleans): =FILTER(A2:D100, (C2:C100="Male")+(C2:C100="Other"))

SORT, UNIQUE & SEQUENCE

SORT / SORTBY / UNIQUE / SEQUENCE
=SORT(A2:D100, 3, -1) -- Sort by 3rd column, descending =SORTBY(A2:D100, C2:C100, 1) -- Sort A:D by values in col C =UNIQUE(C2:C100) -- Distinct values only =SEQUENCE(10) -- 1, 2, 3 … 10 =SEQUENCE(5,1,10,10) -- 10, 20, 30, 40, 50

SUMPRODUCT

SUMPRODUCT
-- Multiply units × price per row and sum: =SUMPRODUCT(B2:B10, C2:C10) -- Conditional count (like COUNTIFS, works in all versions): =SUMPRODUCT((C2:C100="Female")*(B2:B100>60000)) -- Conditional sum (like SUMIFS): =SUMPRODUCT((C2:C100="Female")*B2:B100) -- Weighted average: =SUMPRODUCT(B2:B10, C2:C10) / SUM(C2:C10)

Power Query

Power Query is Excel's built-in data import and transformation engine. Connect to data sources, clean messy data, and reshape it — all recorded as repeatable steps. Access via Data → Get Data.

Import Sources

Loading Data In
From a CSV file: Data → Get Data → From File → From Text/CSV From an Excel Table in the same file: Click inside Table → Data → From Table/Range From a website: Data → Get Data → From Other Sources → From Web From all files in a folder (combine multiple CSVs at once): Data → Get Data → From File → From Folder

Common Transformations

Transformation Steps
-- Rename a column: Double-click the column header → type new name -- Filter rows: Click dropdown arrow → filter by value or condition -- Remove columns: Right-click header → Remove -- Add a calculated column: Add Column → Custom Column → Formula: [Salary] * 0.30 -- Group by (summarise data): Transform → Group By → pick column + aggregate -- Merge two tables (like a JOIN): Home → Merge Queries → pick matching columns → choose join type -- Append (stack two tables): Home → Append Queries → select second table -- Fix data type (text "40000" → number 40000): Click the ABC/123 icon next to the column header
Every step is recorded. The Applied Steps panel lists each transformation. Click any step to preview it, reorder it, or delete it — making your pipeline easy to debug and maintain.

Loading Results

Load & Refresh
-- Load to worksheet: Home → Close & Load → Close & Load To… → Table -- Refresh (pull latest data): Data → Refresh All or right-click query → Refresh -- Auto-refresh on open: Query Properties → Refresh data when opening the file ✓

Conditional Formatting

Conditional Formatting automatically applies colours, icons, or bars to cells based on their values — turning raw numbers into an instantly readable visual layer.

Built-in Rules

Home → Conditional Formatting
-- Highlight cells above a value: Highlight Cell Rules → Greater Than → 70000 → choose colour -- Top / bottom performers: Top/Bottom Rules → Top 10 Items Top/Bottom Rules → Above Average -- Find duplicates: Highlight Cell Rules → Duplicate Values

Data Bars, Color Scales, Icon Sets

Visual Indicators
-- Data Bars: mini bar chart drawn inside each cell: Select range → Conditional Formatting → Data Bars -- Color Scales: heat map gradient: Conditional Formatting → Color Scales → Green-Yellow-Red -- Icon Sets: arrows / traffic lights by threshold: Conditional Formatting → Icon Sets → 3 Arrows

Formula-Based Rules

Custom Formula Rules
-- Highlight entire row if salary > 70000: Select A2:D100 → New Rule → Use a formula =$B2>70000 -- lock column B, let row shift -- Highlight Female employees: =$C2="Female" -- Alternating row colours: =ISODD(ROW()) -- Multiple conditions: =AND($B2>60000, $C2="Female") -- Highlight duplicate names: =COUNTIF($A$2:$A$100, $A2)>1
Rule order matters: Rules run top to bottom — the first match wins. Use Conditional Formatting → Manage Rules to reorder or edit.
